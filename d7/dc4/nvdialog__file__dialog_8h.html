◆ nvd_get_file_location()

NVD_API NvdDynamicString * nvd_get_file_location ( NvdFileDialog * dialog)

Returns the filesystem path chosen through the NvdFileDialog passed.

This function will return the path on the filesystem from the dialog chosen, that you can then use to either open or save the file given. It works with both save and open file dialog types.

See also
nvd_open_file_dialog_new
Parameters
dialogThe file dialog to take the filename from.
savebufA pointer to a buffer where the path will be written, must not be NULL.
Returns
A NvdDynamicString if a file/folder was selected, or NULL if no path was selected by the user.

◆ nvd_open_file_dialog_new()

NVD_API NvdFileDialog * nvd_open_file_dialog_new ( const char * title,
const char * file_extensions )

Creates a new, empty NvdFileDialog to be used for retrieving a file and returns it.

This creates a new file dialog and returns it. In order to use it, you must call nvd_get_file_location and pass a pointer to write the file location to.

Note
This function creates problems with the Adwaita backend. Those problems should be fixed by the time v0.6.0 is officially released.
Parameters
titleA string to put as the dialog title.
file_extensionsUnused legacy parameter, just pass NULL in here.
Returns
An empty NvdFileDialog object if successful, otherwise NULL and an error retrievable through nvd_get_error is set.
